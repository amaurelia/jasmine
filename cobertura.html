Reporte de Cobertura de Código

Esta página muestra cómo generar reportes de cobertura para tus pruebas unitarias. Sigue las instrucciones abajo para generar el reporte real.

Instrucciones para Generar Cobertura

Paso 1: Instalar Dependencias
npm install
Paso 2: Ejecutar Pruebas con Cobertura
npm run coverage
Paso 3: Generar Reporte HTML
npm run coverage-report
Nota: El reporte HTML se generará en la carpeta coverage/

Ejemplo de Métricas de Cobertura

Cobertura de Líneas (Lines)
95%
Porcentaje de líneas ejecutadas por las pruebas
Cobertura de Funciones (Functions)
90%
Porcentaje de funciones ejecutadas por las pruebas
Cobertura de Ramas (Branches)
85%
Porcentaje de ramas condicionales ejecutadas
Cobertura de Declaraciones (Statements)
93%
Porcentaje de declaraciones ejecutadas

¿Qué Significan las Métricas?

Lines Coverage

Mide qué porcentaje de líneas de código fueron ejecutadas durante las pruebas.

Functions Coverage

Indica qué porcentaje de funciones fueron llamadas durante las pruebas.

Branches Coverage

Mide si se probaron todas las ramas de las estructuras condicionales (if/else).

Statements Coverage

Porcentaje de declaraciones de código que fueron ejecutadas.

Objetivos de Cobertura Recomendados:
  • 80-90%: Cobertura excelente para la mayoría de proyectos
  • 70-80%: Cobertura buena, aceptable para desarrollo
  • 60-70%: Cobertura básica, necesita mejorar
  • <60%: Cobertura insuficiente, alto riesgo

Recursos Adicionales

Herramientas de Cobertura:
  • NYC/Istanbul: Para JavaScript/Node.js
  • Karma: Para pruebas en navegador
  • Jest: Framework todo-en-uno
Comandos Útiles:
  • npm test - Ejecutar solo pruebas
  • npm run coverage - Pruebas + cobertura
  • npm start - Servidor local
