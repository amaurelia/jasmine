📊 Guía de Cobertura de Código

La cobertura de código es una métrica que mide qué porcentaje de tu código es ejecutado durante las pruebas. Este proyecto mantiene altos estándares de cobertura.

🎯 Objetivos de Cobertura

Líneas de Código
80%

Porcentaje mínimo de líneas ejecutadas durante las pruebas.

Funciones
80%

Porcentaje mínimo de funciones que deben ser llamadas en pruebas.

Ramas (Condicionales)
80%

Porcentaje mínimo de rutas lógicas (if/else) que deben ejecutarse.

Declaraciones
80%

Porcentaje mínimo de sentencias ejecutadas.

🔍 Tipos de Cobertura

1. Cobertura de Líneas

Mide cuántas líneas de código se ejecutan. Cada línea debe ser ejecutada al menos una vez.

Ejemplo:
function suma(a, b) { return a + b; }
Cobertura completa = la función se llama al menos una vez.

2. Cobertura de Funciones

Verifica que cada función sea llamada durante las pruebas.

3. Cobertura de Ramas

Garantiza que todas las rutas condicionales (if/else, switch) se ejecuten.

Ejemplo:
if (x > 0) { ... } else { ... }
Necesitas pruebas para ambas condiciones: true y false.

4. Cobertura de Declaraciones

Similar a cobertura de líneas, pero más específica a instrucciones individuales.

📈 Cómo Verificar la Cobertura

Generar Reporte de Cobertura

npm run coverage-report

Revisar Resultados en HTML

Se generará un archivo en coverage/index.html que puedes abrir en tu navegador.

Verificar en Terminal

npm run coverage-check

✅ Mejores Prácticas

  • Siempre prueba tanto casos exitosos como casos de error
  • Prueba todas las ramas condicionales
  • Utiliza pruebas de valores límite (edge cases)
  • Mantén la cobertura por encima del 80%
  • Revisa regularmente el reporte de cobertura
  • No busques 100%, busca significado en las pruebas

🚀 Mejorando la Cobertura

  1. Revisa el reporte HTML de cobertura
  2. Identifica líneas/funciones no cubiertas (líneas rojas)
  3. Escribe nuevas pruebas para esas áreas
  4. Ejecuta cobertura nuevamente hasta alcanzar el objetivo
  5. Commit cuando alcances el target
